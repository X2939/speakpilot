# SpeakPilot Demo Script

建议视频长度控制在 2-4 分钟。录制时必须开声音讲解，并完整展示核心模块。

## 录制前准备

```bash
cd D:\chatgpt\speakpilot
npm run dev
```

打开：

```text
http://localhost:5173
```

浏览器建议使用 Chrome 或 Edge。录制前按 `Ctrl + F5` 强制刷新页面。

## 推荐演示流程

### 1. 项目定位

讲解：

> 大家好，这是 SpeakPilot，一款面向真实场景的 AI 英语口语陪练工具。它不是普通聊天机器人，而是围绕口语学习闭环设计：场景选择、语音或文本输入、AI 角色对话、即时纠错评分，以及课后总结。

操作：

- 展示左侧场景选择。
- 选择「面试」或「旅行」场景。
- 选择「中级」难度。

### 2. 开始场景对话

讲解：

> 用户可以选择面试、点餐、会议、旅行、校园等场景。每个场景都有明确角色和任务目标。进入场景后，AI 会主动发起第一句问题，降低用户开始练习的心理负担。

操作：

- 展示 AI 自动发起的第一句场景问题。
- 如需重新演示，可点击「重新开始」。

### 3. 展示即时纠错

输入：

```text
I dont want to introduce myself, but I can talk about my AI speaking coach project.
```

讲解：

> 这里我故意输入一个真实口语中常见的错误句子。语音输入支持持续监听和自动续听，输入框下方会显示正在监听、自动续听或权限异常等状态。如果浏览器或麦克风环境不稳定，也可以随时切换到文本输入。系统会在保持对话推进的同时，在右侧给出即时反馈。

展示重点：

- `i dont` -> `I don't`
- 拒绝自我介绍时，引导用户改从项目或优势切入
- 短回答时，给出 `I think ... because ...` 或 `One example is ...` 句式
- 总分、流利度、准确性、词汇、发音清晰度评分

### 4. 展示跟读发音评测

讲解：

> 发音评测不是直接对文字打分。这里采用跟读任务：用户朗读 AI 给出的目标句，系统比较目标句和语音识别文本，并结合识别置信度评估发音清晰度。当前是轻量评测方案，后续可以替换为音素级发音模型。

操作：

- 点击「跟读评测」。
- 朗读界面里的目标句。
- 点击「完成评测」。
- 展示跟读分数、文本匹配度和识别置信度。

### 5. 展示稳定反馈模式

讲解：

> 模型层优先调用 OpenAI-compatible API。考虑到免费模型可能限流或超时，后端设计了稳定反馈模式。模型不可用时，本地评分器会接管基础语法、表达和格式纠错，保证比赛现场演示不断链。

注意：

- 不要说“模型挂了”。
- 说“稳定反馈模式保障低延迟反馈和可用性”。

### 6. 再完成 1-2 轮对话

可输入：

```text
I agree with this plan because it can help me learn fast.
```

或：

```text
I want to go to Beijing next week, but I need help with the hotel.
```

讲解：

> 对话区支持多轮练习，消息区有独立滚动条，右侧持续更新最新一句的学习反馈。

### 7. 生成课后总结

操作：

- 点击「生成课后总结」。

讲解：

> 课后总结不是简单生成一段评语，而是基于每轮即时反馈做聚合分析。系统会输出能力画像、错误统计、主要问题、下一步目标和复练任务。

展示重点：

- 能力画像：流利度、准确性、词汇、发音清晰度、场景完成度
- 错误统计：grammar、expression、fluency 等
- 复练任务：下一次具体练什么

### 8. 展示历史摘要

讲解：

> 每次生成课后总结后，系统会自动保存一条历史摘要。这里没有做复杂账号系统，而是用浏览器本地存储保留最近 20 次练习，适合比赛 Demo 展示学习轨迹。后续可以迁移到 SQLite 或用户账户体系。

操作：

- 查看左侧「练习历史」列表。
- 点击刚才保存的历史项。
- 展示历史分数、摘要和原始对话会恢复到页面中。

### 9. 技术架构收尾

讲解：

> 技术上，前端使用原生 HTML/CSS/JavaScript 保持轻量，后端使用 FastAPI 承载场景逻辑、模型调用、评分和总结。前后端通过 HTTP JSON 解耦，模型层通过 OpenAI-compatible 接口抽象，后续可以切换 OpenAI、DeepSeek、OpenRouter 或私有 vLLM 服务。

## 常见录制失误

- 不要只录屏不说话，FAQ 要求用声音完整讲解。
- 不要只展示聊天，要展示右侧即时纠错和课后总结。
- 不要把 Demo 视频链接藏在子目录，后续要放在 README 显眼位置。
- 不要录太长，优先保证 2-4 分钟内讲清楚闭环。
